# 중개사무소 개업·폐업 현황판

전국 부동산 중개사무소의 월별 개업·폐업 추이를 보여주는 정적 웹 대시보드입니다. 서버 없이 GitHub Pages에 무료로 배포할 수 있습니다.

## 폴더 구성

```
├── index.html            # 대시보드 (data.json을 읽어 차트 표시)
├── aggregate.py          # CSV → data.json 집계 스크립트
├── data.json             # 집계 결과 (처음엔 샘플 데이터)
├── snapshots/            # 매월 다운로드한 원본 CSV를 넣는 곳
└── .github/workflows/    # push하면 자동으로 집계 + 배포
```

## 처음 한 번만 하는 설정

1. GitHub에 새 저장소를 만들고 이 폴더 전체를 push합니다.
2. 저장소 Settings → Pages → Source를 **GitHub Actions**로 선택합니다.
3. push하면 자동으로 배포됩니다. 주소는 `https://<아이디>.github.io/<저장소명>/`

로컬에서 미리 보려면 이 폴더에서 `python -m http.server` 실행 후 http://localhost:8000 접속.
(index.html을 더블클릭으로 열어도 내장 샘플 데이터로 화면은 확인할 수 있습니다.)

## 매월 하는 일 (5분)

1. 매월 1일 전후, 브이월드/K-Geo에서 전국 부동산중개업 CSV를 다운로드합니다.
2. 파일명을 `YYYY-MM.csv`로 바꿔 `snapshots/`에 넣습니다. 예: `snapshots/2026-08.csv`
3. `python aggregate.py` 실행 → `data.json` 갱신 확인
4. commit & push → 자동 배포

## 과거 데이터 채우기 (backfill.csv)

스냅샷 비교가 불가능한 과거 구간은 `backfill.csv`에 수치를 넣어 채울 수 있습니다.

```
월,지역,개업,폐업
2026-01,전국,875,1168
2026-01,서울,245,264
```

- 지역명은 `전국`, `서울`(또는 `서울특별시`) 등 짧은 이름도 자동 인식됩니다.
- 자체 스냅샷 집계가 있는 달은 backfill보다 자체 집계가 우선 적용됩니다.
- backfill 구간은 대시보드 표에 * 표시와 회색으로 구분됩니다.
- 출처는 한국공인중개사협회 월간 통계(부동산정책연구원 발간물, 언론 보도)를 권장합니다.

## 집계 방식 (중요)

- **개업**: 각 사무소의 `등록일자` 기준. 스냅샷이 쌓이면 "새로 등장한 등록번호"도 함께 반영해 정확도가 올라갑니다.
- **폐업**: 연속된 두 스냅샷을 비교해, 이전 달에 있던 등록번호가 사라진 경우로 집계합니다.
- **영업 중단**: 상태구분명이 새로 휴업/휴업 연장/업무정지로 바뀐 경우를 집계합니다.
- **영업중 사무소**: 최신 스냅샷에서 상태구분명이 '영업중'인 곳만 셉니다 (휴업·정지 별도 표기).
- 스냅샷이 2개 이상 쌓여야 폐업·영업 중단 수치가 나오기 시작합니다.
- **지역(시도) 판별**: 법정동명 → 지번주소 → 도로명주소 → 법정동코드 순으로 확인해 누락을 최소화합니다.
- 이 방식은 협회 공식 통계와 수치가 다소 다를 수 있습니다.

## 컬럼명이 달라서 오류가 날 때

데이터 출처에 따라 컬럼명이 다릅니다(예: `개설등록번호` vs `등록번호`). `aggregate.py` 실행 시 컬럼을 못 찾으면 실제 컬럼 목록을 출력해 주므로, 파일 상단의 `*_CANDIDATES` 리스트에 해당 이름을 추가하면 됩니다.
